The Suspects · Six Agents in Character

Each of the six suspects is an independent autogen.beta.Agent with its own LLM config, system prompt, and a @tool for looking up its private dossier. They never see the detective's prompt — only their own — so they stay in character.

The Agent is AG2's atomic unit: a self-contained agent with a config, a system prompt, and zero or more tools. We build one per suspect inside build_suspect().

The @tool decorator turns a Python function into something the LLM can invoke. Here, each suspect's query_dossier closes over their own dossier — the suspect cannot reach another suspect's records.

The system prompt teaches each suspect a "compliance ladder": vague answers are free, specific lies are risky, but if the detective names a data source they must call the tool and report the truth. 

from autogen.beta import Agent
from autogen.beta.tools import tool

def build_suspect(profile):
    dossier = profile.dossier

    @tool(name="query_dossier",
          description=f"Query {profile.display_name}'s "
                      f"private records.")
    def query_dossier(source, start_time, end_time):
        rows = dossier.get(source, [])
        return [r for r in rows
                if start_time <= r[0] <= end_time]

    return Agent(
        name=profile.name,
        config=suspect_llm_config(),
        prompt=_render_prompt(profile),
        tools=[query_dossier],
    )
Why it matters Six identical Agent scaffolds, six different personalities. Swap the model in suspect_llm_config() and every suspect upgrades at once.

The Detective · An Agent That Calls Other Agents

The detective is also an Agent — but its tools orchestrate the rest of the cast. ask_suspect is just a Python function that calls another Agent's .ask(), then harvests the events into verified facts.

AG2 makes Agent-to-Agent delegation a one-liner: await suspect.ask(question). The detective's tool wraps that call, then walks the suspect's ToolCallEvent / ToolResultEvent history to record every forced truth as a VerifiedFact.

The detective has four tools: list_suspects, ask_suspect, list_verified_facts, and the terminal accuse. A tight system prompt keeps it looping until it has enough evidence to accuse. 

@tool
async def ask_suspect(name: str, question: str) -> str:
    """Interrogate a suspect; record forced truths."""
    suspect = suspects[name.lower().strip()]
    reply = await suspect.ask(question)   # Agent → Agent

    for ev in await reply.history.get_events():
        if isinstance(ev, ToolResultEvent):
            CASE_MEMORY.add_fact(VerifiedFact(...))

    CASE_MEMORY.add_turn(InterrogationTurn(...))
    return reply.body

return Agent(
    name="detective",
    config=detective_llm_config(),
    prompt=DETECTIVE_PROMPT,
    tools=[list_suspects, ask_suspect,
           list_verified_facts, accuse],
)

CASE_MEMORY · The Shared Notebook

Every fact the detective forces out of a suspect lands in CASE_MEMORY — a tiny process-level singleton with two lists and a pub/sub hook. It is the only place that long-term state lives.

Two record types fill those lists. InterrogationTurn captures the raw Q&A from ask_suspect, while VerifiedFact records a row the suspect's query_dossier tool actually returned — that's the cross-checkable evidence the accuse tool reasons over.

add_turn and add_fact notify subscribers, so the notebook SSE route and the commentator both react in real time without polling. When the detective later calls list_verified_facts, it is reading from this same store. 

@dataclass
class VerifiedFact:
    suspect: str
    data_source: str   # "gps", "keycard", "phone_log"…
    query: dict        # {"start_time": "21:30", …}
    result: Any        # rows the suspect's tool returned
    timestamp: float

@dataclass
class CaseMemory:
    interrogation_log: list[InterrogationTurn]
    verified_facts:    list[VerifiedFact]

    def add_fact(self, fact):

The pattern One Agent's tool is another Agent's .ask(), and a shared in-memory store is the bus that lets the detective, the notebook UI, and the commentator stay in sync without an external broker.

The Commentator · A Side-Channel Agent

The commentator never blocks the investigation. It runs as a background Agent that wakes up whenever CASE_MEMORY records a new fact, then publishes a one-liner to a Server-Sent-Events stream.

It's the same Agent primitive, just wired differently. The commentator subscribes to memory changes; for each new VerifiedFact, the CommentaryEngine seeds a one-shot commentator.ask(seed).

Two read-only @tools let it peek at recent memory. The prompt locks it to one or two punchy sentences — sports-announcer style — so the live feed stays snappy.

The Stage · Streaming, Memory & Time

Three small pieces hold everything together: AG2's AGUIStream (one ASGI route per Agent), our own CASE_MEMORY (the shared notebook), and a game clock that the GameMaster freezes when the verdict lands.

AGUIStream is the AG-UI bridge — wrap an Agent in AGUIStream(agent).build_asgi() and you get a streaming HTTP endpoint emitting TOOL_CALL_START, TOOL_CALL_ARGS, TEXT_MESSAGE_* events the browser renders live.

CASE_MEMORY is a plain Python singleton with an observer hook. The detective writes; the commentator reads; the notebook UI subscribes via SSE. No framework magic — just functions and queues.

GameMaster + GameClock own the rules. accuse() calls GAME_MASTER.finalize(), which freezes the clock and stamps an elapsed time onto the verdict. 

from autogen.beta.ag_ui.stream import AGUIStream

# One ASGI route per Agent — streams AG-UI events
# straight to the browser.
routes.append(Route("/agent/detective",
                    AGUIStream(detective).build_asgi()))
routes.append(Route("/agent/commentator",
                    AGUIStream(commentator).build_asgi()))
for name, agent in suspects.items():
    routes.append(
        Route(f"/agent/{name}",
              AGUIStream(agent).build_asgi()))

# Notebook + clock + commentary stream to the UI:
routes.append(Route("/notebook/stream", notebook_stream))
routes.append(Route("/clock/stream",    clock_stream))
routes.append(Route("/commentary/stream",
                    commentary_stream))

One Turn · End to End

Here is the full path from a click on a directive card to a verified fact landing in the notebook.

  1. 1 The user clicks a directive. The browser POSTs it to /agent/detective.
  2. 2 AGUIStream runs the detective Agent; the LLM picks ask_suspect("eleanor", "...") as its first tool call.
  3. 3 Inside that tool, await suspect.ask(question) spins up Eleanor's Agent. Her LLM sees an invoked question and calls query_dossier.
  4. 4 Eleanor returns. The detective's tool walks her event history and writes a VerifiedFact into CASE_MEMORY.
  5. 5 The memory observer wakes the CommentaryEngine; the commentator Agent emits a one-liner to the SSE feed.
  6. 6 Three browser streams — AG-UI events, notebook SSE, commentary SSE — update the UI live.
  7. 7 Eventually the detective calls accuse(...). GameMaster.finalize() freezes the clock, stamps the elapsed time, and the run ends.
Try it yourself All of this is in app/agents/ and app/server.py. Swap a config, add an Agent, or invent a new tool — the rest of the system follows.
